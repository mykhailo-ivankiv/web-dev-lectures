Перелік лекцій з курсу "веб розробка"

Become a programmer, they said. It'll be fun, they said.

Після вивчення даного курсу студенти зможуть розробляти веб-застосунки. Курсовим проектом можуть виступати спрощені варіанти сервісів, типу facebook, gmail, WordPress.

Хочу зауважити, що акцент робитиметься на фундаментальне розуміння принципів побудови веб-застосунків, та мережевої взаємодії. Конкретні методології, фреймворки, парадигми, розглядатимуться не детально.

1. HTML. Історія винекнення. Проблематика. Принципи стандартизації. Семантика.

2. CSS. Проблематика. Загальні принципи. Синтаксис CSS шляхів.

На даному етапі потрібно, щоб студенти вміли користуватись git. Можливо доцільно буде зробити лекцію про git.

3. CSS. Поняття боксу. Типи боксів. Визначення розмірів. Робота з відступами. Поняття потоку.

4. CSS. Розкладки. Позиціонування.

5. CSS. Розкладки. Флоати. Алгоритм. Канонічний приклад застосування. Розкладки за допомогою флоатів.

6. CSS. Розкладки. Флекси.

7. CSS. Гріди.

8. CSS. Препроцесори. Less. SASS. PostCSS

9. Підсумки. Доступність. Фреймворки. Майбутнє CSS та HTML.

Надалі я не певен, чи студенти встигатимуть. Інтенсивнсть зросте на порядок.

10. JavaScript. Історія. Сфери застосування. Синтаксис.

Тут було б добре розповісти ознайомчу лекцію про будову мереж.

11. Принципи клієнт-серверної взаємодії. HTTP протокол. Використання HTTP сервера на Node.js

12. Вид веб серверів, та принципи їх реалізації. Обмеження HTTP, та як їх подолати. HTTP 2

13. Огляд можливостей бібліотеки Express.js

14. Принципи роботи веб браузера. Завантаження JS в браузері. Деякі історічні проблеми. Модульність.

15. Об’єктна модель документу.

16. Робота з асинхронність. Калбеки. Проміси. Async Await.

17. Клієнт-серверні протоколи обміну даних. Ісротія. REST. Проблематика. Шляхи вирішення. Майбутнє.

18. Архітерктура клієнтських застосунків. MV*

19. Функціональне програмування. Класичні принципи. Реактивність. Огляд бібліотек. (Lodash, Ramda, RxJS)

20. Застосування функціонального підходу. ReactJs. Принципи Flux - Redux архітерктури.

21. Інструментарій. Gulp. Babel. Webpack. Лінтери.

22. Автоматизоване тестування.

23. Continuous Integration.

Необхідність деяких лекцій є дискусійною. Надалі буде перелік тих лекцій, які б доцільно було включити в програму.

a. 7-ми рівнева мережева модель OSI

b. Безпека. Алгоритми шифрування. Ін’єкції. Фішінг.

c. Об’єктно орієнтована парадигма програмування в Javascript.

d. Системи контролю версій. Проблематика. Шляхи вирішення. Git.

e. Авторизація та аутентифікація.

f. Типографіка. Історія розвитку. Tex, LaTeX, Веб типографіка.

g. Візуалізація даних. Бібліотека d3.js

h. Робота з векторною графікою. SVG

i. Робота з картинками. Алгоритми стиснення, та формати картинок

j. Огляд баз даних. Реляційні. Ключ-значення. Документно-орієнтовані. Граф-орієнтовані.

k. Eventual consistency. CAP theorem.

l. Проблеми паралельного редагування даних. Operational Transformation (OT), CRDT

m. Javascript діалекти, та майбутнє. ES4 - ES2017. CoffeeScript. Typescript. ClosureScript. Elm. Web Assembly

n. Анімація в вебі.
